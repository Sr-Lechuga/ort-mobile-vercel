## Resumen diario - 2025-01-27

### Cambios realizados

- **Actualización del schema de validación de Activity**: Se actualizó `Backend/src/2_middlewares/request_schemas/acitivityRequest.schema.js` para alinearse completamente con el modelo de Activity.
  - Se agregó importación de `ACTIVITY_STATUS` desde constants.js
  - Se marcaron como requeridos los campos obligatorios del modelo: `title`, `categories`, `location`, `locationCoordinates`
  - Se eliminó el campo `open` que no existe en el modelo

### Nuevos campos agregados al schema

- **`status`**: Validación para estados de actividad con valores permitidos de `ACTIVITY_STATUS`
- **`location`**: Objeto con validaciones para:
  - `country`: string máximo 25 caracteres, requerido
  - `city`: string máximo 25 caracteres, requerido
  - `address`: string máximo 255 caracteres, requerido
- **`locationCoordinates`**: Objeto para coordenadas geográficas con:
  - `type`: string que debe ser "Point", requerido
  - `coordinates`: array de exactamente 2 números (longitud, latitud), requerido

### Mejoras en validaciones

- **Mensajes de error mejorados**: Todos los mensajes de validación están en español y son más específicos
- **Validaciones de longitud**: Se agregaron validaciones de longitud máxima según el modelo de Mongoose
- **Validación de coordenadas**: Se validó que las coordenadas sean exactamente 2 números para representar [longitud, latitud]
- **Consistencia con el modelo**: El schema ahora refleja fielmente la estructura del modelo de Activity
